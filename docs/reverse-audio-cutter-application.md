# 音频裁剪类逆向成果在“小说转广播剧 app”的落地

## 已落地

1. **完整成片合成**：`public/mobile-pipeline.js` 新增离线 WebAudio 合成，把多段 TTS 片段按顺序拼接为 `000-完整广播剧.wav`。
2. **Android 下载目录保存**：`public/app.js` 会把完整成片放在分段音频之前保存到 `Download/白泽声工坊/<作品名>/`。
3. **剪辑台导出落盘**：双轨剪辑/混音导出后，Android Capacitor 环境直接调用 `BaizeMedia.saveAudioSegments` 保存 WAV；浏览器环境保留普通下载链接。
4. **安全基线**：`android/app/src/main/AndroidManifest.xml` 已把 `android:allowBackup` 改为 `false`。

## 与逆向样本的对应关系

- 样本的可借鉴点是音频工程链路：导入 / 裁剪 / 合成 / 保存 / 分享，不是复制其代码或资源。
- 当前采用 WebAudio + Capacitor MediaStore；长剧后续可升级为服务端 FFmpeg 或 Android 原生流式合成。
