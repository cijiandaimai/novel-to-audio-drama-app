# 小说转广播剧 App 优化执行方案

## 目标

用户上传小说后，系统自动完成：

1. 小说解析与故事骨架生成
2. 场景扩展与广播剧化改编
3. 中文对白润色
4. 最终录制版广播剧剧本
5. 影视级音频提示词分段
6. Doubao-Seed-Audio 1.0 分段音频生成
7. 后台音频拼接与下载

## 优化后的模型分工

| 环节 | 模型 | 职责 |
|---|---|---|
| 项目总控 | GPT | 故事骨架、角色表、剧本版本管理 |
| 扩写 | Gemini | 场景扩展、冲突增强、长文补洞 |
| 中文台词 | 豆包文本模型 | 台词口语化、角色语气、中文情绪表达 |
| 声音生成 | Doubao-Seed-Audio 1.0 | 对白、音乐、环境声、拟音音效一体化生成 |
| 后期处理 | 后台音频引擎 | 音频段保存、拼接、导出 |

## 自动执行流水线

1. **上传小说**
   - 支持文本和 Markdown。
   - 自动按章节或长度切分。

2. **GPT 生成项目资料包**
   - 故事骨架
   - 角色小传
   - 人物关系
   - 不得改动的设定
   - 广播剧改编策略

3. **Gemini 扩展分场大纲**
   - 补充戏剧冲突
   - 标注每场目标和情绪曲线
   - 不传整本小说，只传资料包和必要上下文

4. **豆包优化对白方向**
   - 输出角色语气规则
   - 输出台词风格禁忌
   - 输出每场关键对白建议

5. **GPT 生成剧本 1**
   - 场次编号
   - 场景空间
   - 旁白、对白、音乐、音效
   - 预计时长

6. **豆包二次台词优化**
   - 只改台词
   - 不改剧情结构
   - 不新增重大设定

7. **GPT 生成剧本 2**
   - 最终录制版
   - 明确角色声线、音效时间线、混音要求

8. **生成音频提示词**
   - 使用 `AI影视级音频提示词写作指导.md`
   - 每段控制在 30-90 秒
   - 每条提示词包含前景、中景、背景和混音层级

9. **Doubao-Seed-Audio 1.0 生成音频**
   - 逐段生成
   - 失败段单独重试
   - 保留每段提示词和音频文件

10. **后台合成**
    - 当前原型支持 WAV 段落拼接。
    - 正式版建议接入 FFmpeg 或云端音频处理服务，以兼容 MP3/M4A/WAV。

## 当前原型说明

- 没有配置 API Key 时，系统会走演示模式，生成完整脚本、提示词和静音占位 WAV。
- 配置 GPT/Gemini/豆包文本 API 后，对应阶段会调用真实模型。
- Doubao-Seed-Audio 1.0 当前公开信息显示仍有体验/邀测属性，因此原型提供 HTTP 适配器；正式接口开放后填入接口地址即可替换演示模式。

## 成本与稳定性策略

- 不在每一步传整本小说。
- 将小说拆为章节和上下文包。
- 剧情结构由 GPT 统一管理，减少多模型互相覆盖。
- 音频生成按段执行，失败只重跑当前段。
- 每一步输出都保存到本地，方便继续创作和人工检查。
