# 白泽 IP 预演台

AI 影视预创作工具。用于把灵感、小说、IP 设定和人物关系快速预演成短剧/长剧大纲、剧本、角色对白、广播剧样片和影视化提案。

## 酒馆是什么：AI 角色世界沙盘

酒馆不是普通聊天页，而是一个面向影视预创作的 **AI 角色扮演世界模拟器**。

你可以把它理解成：

> 玩家先搭建世界、规则和主要角色，然后让一群 AI 在这个世界里进行角色扮演、互相碰撞，创造一个可以持续生长的仿真故事世界。

它的目标不是一次性写完剧本，而是在正式进入短剧、长剧、广播剧制作前，先帮创作者验证一个 IP 是否“有戏”。

### 基本逻辑

1. **先设计世界背景**
   - 世界观、时代、地点、势力关系
   - 酒馆规则、禁忌、危险来源
   - 当前故事阶段、长期伏笔、人物关系

2. **再设计主要角色**
   - 角色身份、性格、说话方式
   - 目标、秘密、弱点、行为边界
   - 与其他角色的关系、冲突和筹码

3. **让 AI 扮演角色**
   - AI 按角色卡说话
   - 按世界书行动
   - 承接前面对话和长期记忆
   - 自动制造冲突、选择、反转、误会、合作或背叛

4. **玩家参与或旁观**
   - 可以作为主角进入世界对话
   - 也可以像导演一样推动剧情
   - 还可以通过不同角色反复试戏，收集可改编桥段

### 它适合做什么

- 验证一个故事设定有没有戏剧张力
- 测试角色口吻是否鲜明
- 让角色在同一世界观里自然互动
- 生成短剧/长剧的灵感片段、冲突节点和对白素材
- 把零散想法沉淀成可继续开发的 IP 资料
- 为后续小说改编、广播剧生成、影视化提案搭桥

### 和创作台的区别

- **酒馆**：让故事先“活起来”。重点是角色扮演、世界模拟、剧情试演、灵感收集。
- **创作台**：把故事进一步加工成作品。重点是大纲、剧本、广播剧文本、音频提示词和成品音频。

简单说：

> 酒馆负责预演，创作台负责制作。

### 史官与史书

酒馆里还可以开启“史官”功能。史官不是普通角色，而是这个模拟世界的记录者。

- **史书**：按时间线记录已经发生的世界事实，例如纪年、地点、角色死亡、迁徙、权力变化、重大事件和人物心态转变。
- **不可改写**：史书里的已发生事件会作为后续剧情的重要参考。后续可以引导走向，但不能随意推翻已经发生的事实。
- **史官审校**：检查最近一轮 AI 回复是否 OOC、违反角色卡/世界书/史书、改写已发生事实或出现信息越权。第一版只生成审校报告，不自动改写剧情。
- **独立模型**：史官修史、自动修史和史官审校可以单独选择 API 模型；角色对话仍按每个角色自己的 API 模型回复。
- **必须接入 API**：史官需要模型理解多轮剧情、提炼事实并整理时间线，因此必须把酒馆回复方式切到 API 或自动，并配置可用模型。
- **玩家可扮演史官**：玩家可以用史官身份和 AI 角色沟通，像导演、编年史作者或世界意志一样询问、校准和引导剧情。

### 推荐新手流程

1. 打开酒馆。
2. 先写世界书：背景、规则、势力、禁忌、当前危机。
3. 建 2-5 个主要角色：主角、对手、盟友、隐藏关键人物。
4. 给每个角色写清楚说话方式、目标和秘密。
5. 选择角色开始对话，让 AI 按角色身份推进剧情。
6. 剧情推进后点击“史官修史”，把已经发生的大事沉淀进史书。
7. 对关键回复点击“史官审校”，检查是否 OOC、冲突史书或让角色知道了不该知道的信息。
8. 遇到精彩对白或桥段后，导出 TXT 或转入创作台继续加工。
9. 在创作台里把这些素材整理成短剧/长剧大纲、广播剧脚本或影视化提案。

### 一句话定位

**白泽 IP 预演台 = AI 影视预创作工具。酒馆模块 = 角色世界沙盘，用来让 IP 在正式制作前先跑起来。**

## 功能

- 首页免费有声书/广播剧展示
- 小说上传或粘贴
- GPT / Gemini / 豆包文本 / Doubao-Seed-Audio 1.0 API 配置页
- 自动创作流水线
- 剧本 1、剧本 2、音频提示词和合成音频产物
- 应用内音频播放器
- 自定义播放器背景图
- 角色音色参考上传
- 跳过编剧流程，直接上传音频提示词生成音频
- 创作历史记录
- 后端中转模式，避免正式 APK 暴露付费 API Key

## 启动

```bash
npm start
```

然后打开：

```text
http://localhost:4173
```

## 说明

没有配置 API Key 时，应用会使用演示模式，生成脚本、提示词和静音占位 WAV，便于先验证流程。

Doubao-Seed-Audio 1.0 的正式接口如果仍处于体验或邀测阶段，可以先在配置页使用演示模式；拿到正式接口地址后，填入音频模型配置即可替换。

正式给别人安装使用时，建议部署后端中转，把 Key 放到服务器环境变量里。见：

[docs/backend-relay.md](docs/backend-relay.md)

## 安卓 App

项目已接入 Capacitor，可生成 Android 原生工程：
